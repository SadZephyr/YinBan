# YinBan UI Design Spec

本文件是影伴 / YinBan 最终前端重设计的可执行视觉规范。适用范围为当前 Android 原生工程的 UI 视觉实施，不授权修改业务逻辑、页面跳转、角色值、数据模型、WebSocket、后端或硬件逻辑。

## 1. 项目定位

影伴 / YinBan 是面向轻中度成年 ASD 用户的 AI 辅助独立生活系统。产品目标是帮助用户在日常生活中获得低刺激、可信赖、可理解的陪伴、提醒、连接和紧急求助支持。

产品不是医疗诊断工具，不是儿童陪伴玩具，也不是监控后台。患者端强调自主、稳定和安全感；监护人端强调连接、等待、响应和必要时支持。

## 2. 目标用户

- 患者端：轻中度成年 ASD 用户，可能对强色彩、强动效、拥挤信息和不确定状态敏感。
- 监护人端：家人或照护关系中的成年人，需要快速理解连接状态、实时数据、患者消息、视频画面、通话和 SOS 状态。
- 共同需求：低认知负担、清晰状态、稳定布局、明确触达、少干扰。

## 3. 总体视觉关键词

成熟、低刺激、可信赖、温暖、清晰、克制、成人化、非医疗化、非监控化。

页面应以浅背景、白色卡片、深墨文字、统一紫蓝主色、轻描边和少量阴影组成。不得做成明显 AI 自动生成模板风格。

## 4. 成人化原则

- 用户形象必须是成年人，不使用儿童头像、儿童插画或幼态卡通语气。
- 圆角、卡片和插图可以柔和，但不能玩具化。
- 小影火是品牌标识，不是儿童宠物。
- 功能入口使用统一 VectorDrawable，不使用 Emoji 充当正式功能图标。

## 5. 低刺激原则

- 页面大背景以 `#F7F7FC` 和 `#FFFFFF` 为主。
- 紫色只用于主按钮、选中态、底部导航选中项、少量品牌区域和重点交互。
- 不使用大面积高饱和紫色渐变，不使用强发光、装饰球、星光、无意义玻璃拟态。
- SOS 红色只在危险按钮、危险确认和真正高风险状态中出现，避免长期大面积红色暴露。

## 6. 非医疗化原则

- 避免医院、病患标签、诊断、监护病房等视觉联想。
- 不用十字医院、病床、病例等图标作为角色或状态图标。
- 文案和视觉应表达“支持、连接、独立生活”，而不是“治疗、病患、监测”。

## 7. 非监控化原则

- 监护人端视频画面、实时数据和状态应被组织为“连接与支持”，避免做成深色安防监控界面。
- 视频占位区需要清楚但不压迫，不长期使用纯黑大块制造监控感。
- 患者端不要强化“被监控”感，摄像头、麦克风和位置入口要表现为用户可理解的设备控制与共享。

## 8. 品牌名称规则

- 正式中文名称：影伴。
- 正式英文及仓库名称：YinBan。
- 团队样图中的“影行”不是正式产品名称，最终页面不得出现“影行”。
- 当前代码中的展示文案默认保持原样，包括“AI 影伴系统”“AI 影伴待命中”“患者端”“监护人端”等。
- Codex 不得自行将“AI 影伴系统”缩写或替换为“影伴”。任何品牌展示标题调整必须由用户单独明确授权。

## 9. 开心.png Logo 使用规则

当前暂定正式 Logo：`前端/“小影火”表情包/开心.png`。

允许用于：

- 登录页品牌区；
- 小影火 AI 页面主视觉；
- 患者首页主状态卡的小尺寸品牌标识；
- 离线或空状态中的克制品牌插图；
- 少量明确体现影伴品牌身份的区域。

使用要求：

- 保持原比例；
- 不拉伸；
- 不裁切主体；
- 不加高饱和发光；
- 不加入持续跳动等夸张动画；
- 不处理成儿童宠物；
- 不处理成医疗助手；
- 普通页面建议 32dp 至 64dp；
- 登录页或 AI 主视觉最多约 96dp；
- 不在每张卡片和每个按钮中重复使用。

其他表情图当前不得由 Codex 自行加入页面，只能作为未来状态插图候选。

## 10. 正式功能图标规则

消息、位置、通话、摄像头、麦克风、设置、返回、设备、SOS 等正式功能图标必须使用统一 VectorDrawable。

要求：

- 24dp 基准尺寸；
- 统一视觉线宽；
- 统一圆角感；
- 支持 tint；
- 与文本和按钮状态色联动；
- 不使用 Emoji；
- 不使用 `开心.png` 代替功能图标；
- 不使用其他表情包图片代替功能图标；
- 不新增第三方图标依赖。

## 11. 最终色彩规范

以下为当前 YinBan 最终 UI 色彩规范，不再是候选值。Codex 不得自行调整 HEX。

| 语义 | HEX | 用途 |
| --- | --- | --- |
| primary | `#5B5CEB` | 主按钮、选中态、底部导航选中项、重点交互 |
| primary_container | `#EEF0FF` | 选中背景、浅紫卡片底、弱强调 |
| background | `#F7F7FC` | 页面大背景 |
| surface | `#FFFFFF` | 主卡片、弹窗、输入面 |
| surface_variant | `#F1F2FA` | 输入框、空态容器、次级底色 |
| text_primary | `#101426` | 标题、正文、核心数据 |
| text_secondary | `#73758A` | 说明、辅助状态、次级信息 |
| border | `#E2E5F2` | 卡片、输入框、分割线 |
| status_connected | `#20A66A` | 在线、已连接 |
| status_waiting | `#D98200` | 等待患者、等待连接、处理中 |
| status_offline | `#8A8FA3` | 离线、断开、不可用 |
| status_warning | `#F2A51A` | 温和提醒、注意 |
| status_danger | `#D92D2D` | SOS、危险确认、高风险 |

强制规则：

- 不创建第二套主题；
- 不新增其他品牌紫色；
- 不修改以上 HEX；
- 不新增未经定义的强调色；
- 不增加大面积高饱和紫色渐变；
- 在线、等待、离线必须分别使用对应语义色；
- 状态栏不得继续使用整块高饱和紫色背景。

## 12. 字体和信息层级原则

- 页面标题：28sp，Semibold，适合登录标题、页面主标题。
- 模块标题：20sp，Semibold，适合卡片标题和分区标题。
- 正文：16sp，Regular，适合主要说明、列表内容。
- 辅助说明：14sp，Regular，适合状态补充、帮助说明。
- 状态文字：13sp，Medium，适合连接、离线、等待标签。
- 按钮文字：17sp，Semibold，主按钮居中。
- 数据信息：26sp，Bold，适合实时数据核心数值。

不得为了视觉效果修改业务文案。允许调整字号、行高、换行、控件宽度、间距和文字位置。

## 13. 圆角原则

- 小圆角：8dp，用于标签、轻量状态、局部小控件。
- 中圆角：14dp，用于按钮、输入框。
- 大圆角：18dp，用于主要卡片。
- 弹窗圆角：16dp。
- 底部导航和大容器不使用夸张圆角。
- 不得每个页面使用不同圆角体系。

## 14. 描边原则

- 普通卡片描边：1dp，`border`。
- 选中卡片描边：1.5dp 至 2dp，`primary`。
- 输入框描边：1dp，聚焦时使用 `primary`。
- 危险态描边：`status_danger`，仅用于 SOS 或危险确认。

## 15. 阴影原则

- 以轻阴影为主，配合描边提供层级。
- 登录主卡、页面主卡可使用低强度阴影。
- 不使用强投影、悬浮球式阴影、发光阴影。
- 阴影不得替代状态表达。

## 16. 留白与页面密度原则

- 页面水平边距建议 20dp。
- 卡片内边距建议 20dp。
- 区块之间使用 16dp 或 24dp 间距。
- 患者端保持更安静的阅读节奏，监护人端可略高密度但不得拥挤。
- 空状态需要有清楚说明，不让页面出现无意义大空白。

## 17. 状态表达原则

状态必须通过“语义色 + 文案 + 位置稳定”表达，不得全部使用紫色。

- 已连接：`status_connected`。
- 等待：`status_waiting`。
- 离线/断开：`status_offline`。
- 温和提醒：`status_warning`。
- SOS/危险：`status_danger`。

状态栏应为浅底、轻描边或小圆点结构，不使用长期高饱和紫色整块背景。

## 18. SOS 和风险表达原则

- SOS 必须醒目，但红色面积应受控。
- 患者首页 SOS 可以保留强入口，但优先用清晰图标、文字和边界表达，不长期铺满大红块。
- SOS 确认弹窗必须保持二次确认、取消、确定求助和共享位置说明，不得改文案或流程。
- AI 风险提示和跨端风险消息不得改逻辑和文案。

## 19. 动效原则

- 动效只用于状态过渡、按钮反馈、轻量加载。
- 不使用持续跳动、闪烁、强缩放、炫光扫过。
- 小影火 Logo 不做夸张动画。
- 状态变化应稳定、可预测，避免增加焦虑感。

## 20. 视频流功能保护原则

视频流是本轮 UI 改造中的最高风险功能之一。视频是功能组件，不是装饰图片，也不是可自由替换的静态预览区。监护人视频区域和通话页不得被当成普通静态页面重新实现。

当前真实视频链路必须保持原样：

```text
硬件摄像头服务
-> 后端 WebSocket
-> device_control_request / stream_start / stream_stop
-> stream_url / stream_status
-> Android MjpegStreamer
-> ImageView 显示实时 MJPEG 画面
```

不得将该链路描述或改造为 WebView、系统播放器、SurfaceView、TextureView 或其他视频实现。UI 改造不得改变视频传输、控制和显示链路。

视频卡片可重绘，但内部真实显示控件不得替换：

- `iv_video` 必须继续作为监护人首页真实 MJPEG 画面 ImageView；
- `iv_video_stream` 必须继续作为通话页真实 MJPEG 画面 ImageView；
- `tv_guardian_video_placeholder` 必须继续承担监护人视频占位、加载、错误显示；
- `btn_guardian_view_video` 和 `btn_sos_view_video` 必须继续触发原请求画面逻辑；
- `switch_video` 和 `switch_audio` 必须继续触发原设备控制入口。

以下文件在整个 UI 实施阶段默认冻结，禁止修改：

- `app/src/main/java/com/yinban/ai/ui/GuardianActivity.kt`
- `app/src/main/java/com/yinban/ai/ui/VideoCallActivity.kt`
- `app/src/main/java/com/yinban/ai/ui/PatientActivity.kt`
- `app/src/main/java/com/yinban/ai/hardware/MjpegStreamer.kt`
- `app/src/main/java/com/yinban/ai/hardware/HardwareStreamManager.kt`
- `app/src/main/java/com/yinban/ai/network/WebSocketManager.kt`
- `app/src/main/java/com/yinban/ai/network/MessageModels.kt`
- `app/src/main/java/com/yinban/ai/storage/PreferenceManager.kt`
- `后端/main.py`
- `硬件/main_visual_service.py`
- `app/src/main/AndroidManifest.xml`

不得修改：

- MJPEG 地址解析；
- `stream_url`；
- `stream_status`；
- `camera_status`；
- `device_control_request`；
- `stream_start`；
- `stream_stop`；
- WebSocket 消息类型和字段；
- 视频启动、停止和页面生命周期处理；
- 设备角色和房间逻辑。

如果未来发现视觉调整确实必须修改上述 Kotlin、后端、硬件或 Manifest 文件，必须停止当前实施，将其列为单独高风险任务，获得用户明确授权后才能修改。不得由 Codex 自行判断。

构建成功不等于视频功能通过。视频必须经过真实设备服务验收：后端、硬件 `main_visual_service.py`、患者端、监护人端进入同一房间后，验证请求画面、`stream_url`、placeholder 切换、`iv_video` / `iv_video_stream` 真实画面显示和停止流逻辑。

允许的 XML 视觉调整仅限外层：

- 在视频区域外增加统一卡片容器；
- 调整外层卡片圆角、背景、描边和边距；
- 调整视频区域合理高度；
- 调整 placeholder 的字体、图标和排版；
- 调整请求画面按钮颜色、圆角和尺寸；
- 调整视频容器与其他模块之间的间距。

但必须确保：

- `iv_video` 和 `iv_video_stream` 最终测量宽高均大于 0；
- 不被其他控件遮挡；
- 不因错误约束移出屏幕；
- 不因 `alpha=0` 或错误 visibility 导致不可见；
- 不被纯色背景覆盖；
- 不因父容器高度为 0 而黑屏；
- 横纵比例适合 1080x2400；
- 占位层隐藏后真实画面完整可见；
- `scaleType` 的改变不会裁掉主要画面；
- 视频显示区域不能退化为过小缩略图。

## 21. 功能和业务红线

必须保留：

- 登录角色选择、账号密码、服务器 IP、配对码、快速体验、三步流程；
- patient / guardian 角色值；
- 患者端首页、小影火 AI、监护栏、我的；
- 摄像头、麦克风、位置、设备状态、SOS、消息、通话、画面、AI 对话；
- 连接、等待、离线、错误和空状态；
- 监护人首页、实时数据、患者消息、视频画面、请求画面、消息、音频通话、SOS 状态；
- 离线聊天页、SOS 确认弹窗、音频通话页、设置和退出登录。

禁止：

- 删除、隐藏、合并或弱化最新版功能；
- 修改点击行为、显示条件、页面跳转或数据流程；
- 修改 WebSocket、消息类型、字段、AI、SOS、视频、位置、设备控制、后端或硬件逻辑；
- 根据样图新增事件记录、风险分析、支持者管理等不存在功能；
- 把“患者端”改为“用户端”；
- 把“监护人端”改为“支持者端”；
- 修改业务文案，除非用户明确授权。

## 23. 明确禁止的视觉方向

- Emoji 功能图标；
- 儿童化卡通；
- 医院或病患标签式视觉；
- 监控后台感；
- 大面积高饱和渐变；
- 强发光；
- 无意义玻璃拟态；
- 装饰球或星光；
- 每个页面使用不同圆角；
- AI 自动生成模板感；
- 在页面中频繁堆放小影火。
